Unmanic can be configured with multiple libraries.

A default library will be created on the first install. This will be given a randomly generated name.
All aspects of this default library may be modified, including its name, but the library itself can never be removed.

To add a new library, click the <img className={"inline-text-screenshot"} src={require('/img/unmanic-add-to-list-icon.png').default} title={"Add"} /> button
under the libraries list.

<img className={"screenshot"} style={{ width: '400px' }} src={require('/img/unmanic-settings-library-add.png').default} />

Select the directory that you wish to use for the new library.

:::tip
You may add multiple libraries that point to the same path.
This is commonly used for configuring custom plugin stacks to suit the needs of multiple file types within a directory.
:::

<img className={"screenshot"} style={{ width: '400px' }} src={require('/img/unmanic-settings-library-add-initial-path-select.png').default} />

A new library will be created with a blank configuration.

<img className={"screenshot"} style={{ width: '400px' }} src={require('/img/unmanic-settings-library-added.png').default} />
